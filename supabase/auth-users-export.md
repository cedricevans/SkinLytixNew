## How to Import Auth Users into a New Supabase Project

You **cannot** directly INSERT into `auth.users`. Instead, use the **Supabase Admin API** with the service role key:

### Option 1: Supabase Admin API (Recommended)

Use `supabase.auth.admin.createUser()` to create each user with their original UUID:

```javascript
const { createClient } = require('@supabase/supabase-js');

const supabase = createClient(
  'https://YOUR-NEW-PROJECT.supabase.co',
  'YOUR-SERVICE-ROLE-KEY' // NOT the anon key
);

const users = [
  { id: '4efb5df3-ce0a-40f6-ae13-6defa1610d3a', email: 'alicia@xiosolutionsllc.com', display_name: 'Adupass' },
  { id: '1efb1396-aa1e-419c-8ba2-1b6366143783', email: 'support@skinlytix.com', display_name: 'Test User' },
  // ... add all users here
];

async function importUsers() {
  for (const user of users) {
    const { data, error } = await supabase.auth.admin.createUser({
      id: user.id,           // Preserves the original UUID
      email: user.email,
      email_confirm: true,   // Mark as confirmed
      user_metadata: { display_name: user.display_name },
      password: 'TempPassword123!' // Users will need to reset
    });
    
    if (error) {
      console.error(`Failed to create ${user.email}:`, error.message);
    } else {
      console.log(`Created: ${user.email}`);
    }
  }
}

importUsers();
```

### Option 2: SQL (Direct DB access only)

If you have direct PostgreSQL access to the new project:

```sql
-- This requires superuser/service role access
INSERT INTO auth.users (id, email, encrypted_password, email_confirmed_at, created_at, updated_at, raw_user_meta_data)
VALUES (
  '4efb5df3-ce0a-40f6-ae13-6defa1610d3a',
  'alicia@xiosolutionsllc.com',
  crypt('TempPassword123!', gen_salt('bf')),
  now(),
  '2025-10-04 19:45:05.302864+00',
  now(),
  '{"display_name": "Adupass"}'::jsonb
);
-- Repeat for each user...
```

### ⚠️ Important Notes

1. **Passwords cannot be migrated** — users will need to use "Forgot Password" to set new ones
2. **UUIDs must match exactly** — all foreign keys in `profiles`, `user_analyses`, etc. depend on these
3. **Create auth users FIRST**, then import `profiles` and other tables
4. **The `handle_new_user` trigger** will auto-create profile rows, so either:
   - Disable the trigger before import, then import profiles manually
   - OR let the trigger create profiles, then UPDATE them with the exported data
